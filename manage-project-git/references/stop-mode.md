# 停止模式

用於機密、版本差異、衝突、失敗或其他需要使用者決策的情況。進入後，在使用者選擇方案且安全範圍重新確認前，不得改變 Git 狀態。

## 觸發條件

- 疑似憑證、token、私鑰、個資、內部端點、商業規劃或其他不公開內容。
- 既有 staged 內容混有本次範圍外檔案。
- detached HEAD、進行中的 merge／rebase／cherry-pick／revert／bisect、未解衝突或目標分支不明。
- 重要測試、lint、build、hook 或 Skill 驗證失敗。
- remote／upstream 不符、本機落後、雙方分歧或遠端已有不相關歷史。
- fetch、認證、網路、LFS、保護分支、權限、non-fast-forward、hook 或 push 錯誤。

## 立即處置

1. 停止暫存、commit、push 與任何擴大影響的動作，保留工作樹、index、branch、remote 與歷史。
2. 不得自行 unstage、刪除、`clean`、pull、merge、rebase、修改 remote、跳過 hook、改寫歷史或強制推送。
3. 收集足以決策的最小證據，不輸出完整機密值或不必要敏感內容。

## 機密處理

- 只回報檔案、位置範圍與風險類型，不顯示完整值。
- 未提交內容優先建議移除、改用環境變數／機密管理並補充忽略規則。
- 已進入 commit 或遠端時，說明 `.gitignore` 無法清除歷史；優先撤銷或輪替，再提出歷史清理方案。
- 未經明確同意不得改寫歷史或強制推送。

## 版本、錯誤與選項

- 本機落後：提供取得資訊、rebase、merge、新分支／PR 或暫不處理的取捨。
- 雙方分歧：說明兩側 commit 與協作者影響，再提供 rebase、merge 或新分支方案。
- 不相關歷史或 remote／upstream 不符：列出目前與預期值，提供更正 remote、新分支／儲存庫或經確認整合的方案。
- 驗證、hook、認證、權限或網路錯誤：回報失敗階段、精簡錯誤與安全修復選項，不得以 `--no-verify` 繞過。
- 衝突：列出檔案與操作狀態，不猜測內容取捨。

只有使用者明確要求改寫精確遠端分支並接受影響後，才可考慮 `--force-with-lease`；不得使用裸 `--force`。

## 回報與恢復

1. 回報 branch、HEAD、remote、ahead／behind、工作樹及已完成動作。
2. 說明精簡錯誤、衝突或風險類型與可能受影響的檔案、commit、遠端及協作者。
3. 提供安全選項、主要取捨、推薦方案及暫不處理，明確等待使用者選擇。
4. 使用者只核准一項時，只執行該項。
5. 修復或整合後，重新確認工作樹、index、HEAD、branch、remote 與風險模式，再載入適用模式並重跑受影響檢查。
